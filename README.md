Business Performance Analytics Dashboard

Problem Statement:
Analyze transactional sales data to evaluate revenue performance, customer ordering behavior, and product contribution trends.

Database Design:
Designed a relational schema with
Customers Table
Products Table
Orders Table
Order_items Table
Established one-to-many relationships using foreign key constraints.

SQL Analysis:
Revenue generated by each customer.
Product revenue ranking using DENSE_RANK.
Repeat customer identification.
Month-over-month revenue growth using LAG.

PowerBI Dashboard:
KPI cards (Revenue, Orders, Repeat Customers, AOV).
Revenue trend over time.
Revenue by region.
Top products by revenue.
Interactive slicers (Region, Category, Date).

Business Insights:
Revenue concentration varies by region.
Certain products contribute disproportionately to revenue.
Repeat customers form a smaller but valuable segment.
Revenue trends fluctuate across months.

Tools Used
MySQL | Power BI | SQL
